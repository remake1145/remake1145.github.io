    # G题意

    每一轮操作：捕食残骸获得A0mod7A_0~mod~7A​0​​mod7，清空残骸后增添两个新残骸： A_1=(213346089\times A_0+870413)mod2^{32}\ A_2=(166042049\times A_0+598777)mod2^{32} 给定初始残骸seedseedseed和目标能力值SSS大小，求最小轮数。

    # G代码

    #include <bits/stdc++.h>
using namespace std;

#define ll long long int

ll mod = 4294967296;
ll f1 = 213346089, f2 = 870413;
ll s1 = 166042049, s2 = 598777;

int first[8] = {5, 3, 1, 6, 4, 2, 0}, second[8] = {4, 2, 0, 5, 3, 1, 6};
ll seed;
int S;

int limit = 1;

bool dfs(int d, int total, ll pre) {
//	cout << d << ' ' << total << ' ' << pre << ' ' << pre % 7ll << endl;
	if (d >= limit) {
		return total >= S;
	}
	if (6 * (limit - d) + total < S) {
		return false;
	}
	ll k = (f1 * pre + f2) % mod;
	ll kkk = (s1 * pre + s2) % mod;
	return dfs(d + 1, total + k % 7ll, k) ||
	       dfs(d + 1, total + kkk % 7ll, kkk);
}

int main() {
	scanf("%d%d", &seed, &S);
	int s = seed % 7ll;
	for (;; ++limit) {
//		cout << limit << endl;
		if (dfs(1, s, seed)) {
			break;
		}
	}
	cout << limit << endl;
	return 0;
}

    # W题意

    给定n个字符串,求各个字符串在文本串中的出现次数。

    # W过程

    AC自动机构造： 1.先建个trietrietrie树，带个根节点(不表示任何状态)。可以发现，trietrietrie树中每个节点都代表着n个字符串其中某个（某些）字符串的一个前缀。 trietrietrie树构建的时候可顺带建个endendend表示每个字符串结束位置(本题)。

    failfailfail指针：匹配所有模式串前缀中当前状态的最长后缀，如she，he两个string，she中e的failfailfail指针指向e。

    构造failfailfail过程：设当前节点nownownow. 假设深度小于uuu的节点的failfailfail指针已经求得（所以搜索过程中便用BFS罢）：

    1. 若trie[now][c]trie[now][c]trie[now][c]存在，fail[trie[now]][c]=trie[fail[now]][c]fail[trie[now]][c]=trie[fail[now]][c]fail[trie[now]][c]=trie[fail[now]][c],trie[now][c]trie[now][c]trie[now][c]入队列。

    2. 否则，trie[now][c]=trie[fail[now]][c].trie[now][c]=trie[fail[now]][c].trie[now][c]=trie[fail[now]][c].

    普通ac自动机似乎是要不断跳的（whilewhilewhile循环），这里采取这样的原因为： 每一个节点代表一个字符串，trie[now][c]trie[now][c]trie[now][c]表示在now代表的string后加一个字母c变成另一个状态。若该状态存在，则可以将其指向trie[fail[now]][c]trie[fail[now]][c]trie[fail[now]][c],同时也保证了failfailfail指向的后缀的最长性———fail[now]fail[now]fail[now]指向后缀，转变后的failfailfail也是对应新字符串后缀。（构建failfailfail时是不断增长一个长度的增长）

    之后，若在此位置失配，则可以用trietrietrie数组记录位置，便可节省时间。可以观察到，这与并查集路径压缩很像。

    # manacher

    # X题意

    给定一个字符串，求其前后缀（不可重合且可为空串）拼接后最长的回文串长度。

    # X过程

    manacher记录数组d,d[i]表示i处最长回文半径。 输入字符串后，先对其进行扩充，每个原字符串元素的前后都加上'#'(非原字符串元素)以方便奇偶长度回文串的讨论。第一个元素即'#'为下标1. 之后，初始化l=0，r=-1(最靠右子回文串，或者说拥有当前r_max的字符串)，不断进行拓展：对于下一个i，若i在[l,r]之外，则i>r,用朴素算法拓展求解di. 否则，i≤ri\leq ri≤r,则可以发现，在回文串的左半部分存在对称的位置j=l+r-i,则若对于i而言的d[i]满足\leq r-i+1，则可以直接与d[j]相等，否则，由于当前最长回文串的限制，左右不再对称，所以应当截断长度，更新l，r.

    回到本题，若原字符串中前后缀存在相等长度的字符串翻转相等的状况，则无论如何拼接，这部分都可以在拼接后的字符串中成为回文串的一部分，因此可以舍去。之后通过manacher来对剩余字符串求能够到达最左边或最右边的字符串的最长长度，即可解出。
