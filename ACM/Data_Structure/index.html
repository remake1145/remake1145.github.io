# st表 异或

# A题意

共有n个罐头，每个罐头坚固度aia_ia​i​​;m个钥匙，型号为bib_ib​i​​,刚好能打开坚固度bib_ib​i​​的罐头，可以对罐头不断进行⌊x/d⌋\lfloor x/d\rfloor⌊x/d⌋操作,查询区间[l,r][l,r][l,r]的罐头，打开最多罐头前提下最少需要多少钥匙。

# A过程

首先注意到数据规模，开longlong。

对钥匙进行排序+离散化，因为重复的钥匙并不会怎么影响。 之后对每个钥匙进行编号（存map），用一个二重循环不断寻找最小的钥匙，为什么找最小？因为若有一个罐头在进行操作后能被多把钥匙打开，那么最小的钥匙就能打开所有的罐头，因此可以把其他大的钥匙直接丢了。 需要注意的就是若有一个钥匙型号是0那就直接只用一个就行了，完全可以特判。 上面的过程已经为st表准备好了基础，直接用或运算建立st表，而st表的思想。。。图论LCA的题用了，就是不断划分2的幂次的区间罢。 之后对每次的查询，查询出后进行二进制中1的数量的统计即可。

# 二叉搜索树 splay

# D题意

实现一个数据结构，能够完成插入数，删除数，查询xxx数排名，查询排名为xxx的数，查询x的前驱和后继。

# D过程

初看二叉搜索树，然后写了个普通的结果tle了，悲。（源代码仍然保留着被注释后的普通二叉树） 之后明白了普通的二叉搜索树是很容易被卡退化成链的，所以需要维护一个平衡树。常见的有treaptreaptreap和splaysplaysplay，这里用了splaysplaysplay方法。

splaysplaysplay：不断旋转至根节点，同时满足二叉搜索树的性质（左儿子及其子树中节点必小于该节点，右儿子极其子树中节点必大于该节点），并且不会退化为链。

旋转：不断将某个节点上移。分为左旋，右旋，而旋转具体步骤，以右旋为例： 旋转节点为xxx，父亲为yyy。

  1. 将yyy左儿子替换为xxx右儿子，并且原来的xxx右儿子更新父亲。
  2. 更新后，将xxx的右儿子替换为yyy，并更新yyy的父亲。
  3. 若yyy有父亲zzz，则在原来yyy的位置更新位置信息。

splaysplaysplay每访问一个节点都需要旋转到根节点，而旋转过程根据xxx父亲是否为根节点、xxx与父亲的儿子类型相同与否进行不同的旋转。

接下来阐述题意所需操作：

插入：

  1. 若树是空树，直接更新退出。
  2. 若当前节点权值等于kkk，增加当前节点大小，更新信息，并splaysplaysplay.
  3. 否则，不断根据二叉搜索树性质寻找，找到后插入，再splaysplaysplay.

查询：可以直接根据二叉搜索树性质不断查找，并且需要在之前操作时维护sizsizsiz大小，并且不断查找从而得出答案。

查询排名为xxx的数：根据二叉搜索树性质，若左子树非空且甚于排名不大于左子树大小，向左子树查找，否则用sizsizsiz和cnt(u)cnt(u)cnt(u)(该节点的大小)减去k，若小于等于0则返回u权值，否则向右子树查找。

查询前驱后继：需要注意的是，查询的数xxx可能不在二叉搜索树里，所以要进行判断。 这里以前驱为例：先将xxx插入，插入后x便已splaysplaysplay到了根节点，然后通过查找是否有前驱(是否有左子树)，然后去不断递归寻找xxx左儿子的最大值(不断查询xxx左儿子的右子树),然后返回答案.

# I题意

给定一张n个点的无向图，动态操作：添加一条a，b的边，或者询问a,b是否在同一个连通块内。

# I过程

用并查集捏。 若添加a，b，那么若a，b不属于同一个子树，连通后在并查集中都有共同的祖先，可以合并，若属于同一个子树，则可以直接退出（没必要操作力），之后查询时直接查询是否有共同祖先即可。 简单做法，但是卡时间复杂度。所以在实际过程中需要用路径压缩来减少查询次数，用子树大小（点的数量）来按秩合并。最后的最后，也是tle这么多次的原因，是快读😭我没看到快读😭快读加上就好了，令人感叹。

# 分块

# L题意

给定n个序列，n次操作，每次操作询问区间的最小众数。

# L过程

序列排序后分块，离散化。

之后计算块i,ji,ji,j的最大出现次数，若有多个众数存最大众数。

再之后，查询时先对范围进行最小化min(r,block[l]∗blo)min(r,block[l]*blo)min(r,block[l]∗blo)，在l 最小化范围中寻找最大众数，若若区间l，r不在同一个块则对不在块中的数[(block[r]−1)∗blo,r][(block[r]-1)*blo,r][(block[r]−1)∗blo,r]进行查询最大众数,最后输出结果即可。
